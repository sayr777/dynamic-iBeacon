🔐 Алгоритм идентификации

AES-128 — симметричный блочный шифр

Стандарт: NIST FIPS 197 (Advanced Encryption Standard, 2001). Ключ — 128 бит (16 байт), блок — 128 бит (16 байт), 10 раундов операций SubBytes / ShiftRows / MixColumns / AddRoundKey.

Режим работы: ECB (Electronic Codebook). Обоснование: каждый вызов шифрует ровно один блок. IV не нужен — уникальность обеспечивается парой tagID + slot. Безопасность ECB эквивалентна AES при шифровании одного блока.

Совместимость: NIST SP 800-38A (режимы работы блочных шифров). Тест-векторы: NIST CAVP AES.

Схема v2 — два вызова AES на слот

Входной блок (16 байт) для каждого из двух вызовов:

 байт  поле       значение
 ────  ─────────  ──────────────────────────────────
 [0]   TAG_ID Hi  старший байт идентификатора метки
 [1]   TAG_ID Lo  младший байт идентификатора метки
 [2]   slot  b3   (slot >> 24) & 0xFF
 [3]   slot  b2   (slot >> 16) & 0xFF
 [4]   slot  b1   (slot >>  8) & 0xFF
 [5]   slot  b0    slot        & 0xFF
 [6]   variant    0x00 = UUID  │  0x01 = adv
 [7…15] padding   0x00 × 9

slot = unix_time / SLOT_DURATION  (300 с в production, 10 с в прототипе)

Вывод AES и поля iBeacon

 variant = 0x00  →  UUID
 AES(KEY, block0)[0..15] → UUID [16 байт]
 Передаётся в поле Proximity UUID пакета iBeacon.

 variant = 0x01  →  Major + Minor + MAC
 out = AES(KEY, block1)
 Major = (out[0] << 8) | out[1]          ← 16 бит
 Minor = (out[2] << 8) | out[3]          ← 16 бит
 MAC[0] = out[4] | 0xC0                  ← биты 46-47 = 11 (BLE Random Static)
 MAC[1..5] = out[5..9]                   ← 5 байт

BLE Random Static Address: стандарт Bluetooth Core 6.0, Vol 6, Part B, §1.3.2.

BLE Privacy — ротация RadioMAC

Помимо MAC в payload, SoftDevice S140 автоматически меняет RadioMAC (адрес, видимый сниффером) через sd_ble_gap_privacy_set() каждый слот.

Тип адреса: BLE_GAP_ADDR_TYPE_RANDOM_PRIVATE_NON_RESOLVABLE. Стандарт: Bluetooth Core 6.0, Vol 3, Part C, §10.7.

Идентификация на сервере — псевдокод

slot_now = unix_time() // SLOT_DURATION

for s in [slot_now-1, slot_now, slot_now+1]:
  for tag_id in 0..NUM_TAGS:
    b0 = tag_id[2] ‖ s[4] ‖ 0x00 ‖ 0x00[9]
    uuid_c = AES128(KEY, b0)
    if uuid_c != uuid_rx: continue   # быстрое отсечение

    b1 = tag_id[2] ‖ s[4] ‖ 0x01 ‖ 0x00[9]
    out = AES128(KEY, b1)
    if out[0:2]==Major AND out[2:4]==Minor:
      return tag_id                  # ✅ найдено

10 000 меток × 3 слота = 60 000 операций AES-128 ≈ < 1 мс на современном сервере.

Свойства безопасности

Неотслеживаемость: без ключа наблюдатель не может связать пакеты одной метки между слотами — UUID, Major, Minor, MAC и RadioMAC меняются каждые 5 мин.

Стойкость: инвертировать AES-128 без знания KEY — вычислительно неосуществимо при текущем уровне техники (2128 операций перебора).

Общий ключ: один KEY на регион. Для изоляции компрометации используйте иерархию: device_key = AES(master_key, tag_id || 0[14]).
