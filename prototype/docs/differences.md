# Отличия прототипа от производственной метки

## Сравнительная таблица

| Критерий | ProMicro NRF52840 (прототип) | [YJ-16013](../../specs/YJ-16013-datasheet.pdf) nRF52832 (производство) |
|---|---|---|
| **Цель** | проверка алгоритма, отладка | серийное изделие |
| **SoC** | `nRF52840` | `nRF52832` |
| **Flash / RAM** | 1 MB / 256 KB | 512 KB / 64 KB |
| **Размер платы** | 18 × 33 мм | ~20 × 10 мм (модуль) |
| **Питание** | USB-C или JST LiPo | ER14505H (AA, Li-SOCl₂) |
| **Напряжение питания** | 3.3 В (внутр. LDO) | 3.0 В (MCP1700) |
| **LiPo зарядник** | ✅ `LTH7R` на плате (20 µА standby) | ❌ нет |
| **Средний ток** | **~23–25 µА** | **~5.2 µА** |
| **Ресурс (LiPo 500 мАч)** | ~500 дней | N/A (первичная батарея) |
| **Ресурс (ER14505H)** | N/A | ~20 лет |
| **Прошивка** | USB UF2 (drag-and-drop) или SWD | SWD (J-LINK / nRF52 DK) |
| **Программатор** | не нужен (USB UF2) | J-LINK обязателен |
| **SoftDevice** | `S140` (nRF52840) или `S112` | `S112` |
| **LED индикация** | ✅ `P0.09` — мигает при UPDATE_PARAMS | ❌ нет |
| **USB serial debug** | ✅ через CDC или RTT | ❌ нет |
| **APPROTECT** | не нужен (прототип) | ✅ обязателен в производстве |
| **Корпус** | нет (открытая плата) | AK-W-70-4 IP67 |
| **Стоимость** | ~$5–8 (плата) + ~$2 (LiPo) | ~$6.95 (полное изделие) |

## Почему прототип потребляет ~23 µА вместо 5 µА

```
Производство (YJ-16013):               Прототип (ProMicro NRF52840):
─────────────────────────               ─────────────────────────────
nRF52832 deep sleep    1.5 µА           nRF52840 deep sleep    1.5 µА
MCP1700 LDO            1.0 µА           LTH7R зарядник        ~20.0 µА ← главная причина
TX (2 с интервал)      2.65 µА          TX (2 с интервал)      2.65 µА
                      ────────                                 ────────
Итого:                ~5.2 µА          Итого:                ~24.2 µА
```

`LTH7R` — LiPo зарядная микросхема на плате. Она потребляет ~20 µА даже  
когда LiPo полностью заряжен и USB не подключён. Для прототипа это нормально.

## Алгоритм — полностью идентичен

Код `tag_app.c`, `beacon_id.c`, `aes128.c` **не меняется**.  
Меняется только платформенный слой (`tag_platform_nrf52840_promicro.c`):

| Функция | nRF52832 (производство) | nRF52840 ProMicro (прототип) |
|---|---|---|
| `tag_platform_init()` | S112 + LFXO + GPREGRET2 | S140 + LFXO + GPREGRET2 + LED |
| `advertise_once()` | S112 API | S140 API (идентичный) |
| `enter_deep_sleep()` | `sd_power_system_off()` | `sd_power_system_off()` |
| `get_unix_time()` | `GPREGRET2` | `GPREGRET2` (тот же регистр ✅) |
| `set_adv_params()` | без LED | **мигает LED** при смене слота |
| APPROTECT | активируется в производстве | **не активируется** |

## Что проверять на прототипе

1. **Алгоритм**: `Major/Minor` меняются ровно каждые 5 мин → LED мигает раз в 5 мин
2. **Сервер**: `server/lookup.py --simulate --tag-id 1` идентифицирует метку
3. **MAC**: меняется вместе с `Major/Minor` → видно в nRF Connect
4. **iBeacon**: смартфон видит UUID, Major, Minor
5. **Ночной режим**: при настроенном `TAG_NIGHT_MODE_ENABLE=1` интервал LED редеет ночью

## Переход от прототипа к производству

| Шаг | Действие |
|---|---|
| 1 | Убедиться что алгоритм работает на прототипе |
| 2 | Заказать [YJ-16013](../../specs/YJ-16013-datasheet.pdf) + компоненты по [`../docs/cost.md`](../docs/cost.md) |
| 3 | Перенести прошивку: заменить `tag_platform_nrf52840_promicro.c` → `tag_platform_nrf52832.c` |
| 4 | Настроить `TAG_ENABLE_APPROTECT=1` перед производственной прошивкой |
| 5 | Собрать PCB, прошить через J-LINK SWD |
