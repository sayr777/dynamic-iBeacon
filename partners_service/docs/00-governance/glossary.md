# Glossary

## Основные термины

- `Partner Service` — облачный сервис доверенного уточнения координат по BLE-событиям.
- `Location Refine API` — основной публичный API сервиса для клиентских приложений партнёров.
- `Sandbox` — изолированный тестовый контур для интеграции, валидации схем и проверки client-side сценариев.
- `RNIS` — доверенный контур, в котором находятся региональные справочники, часть эксплуатационных данных и интеграционные точки с действующей платформой.
- `KMS/HSM` — контур защищённого хранения и ротации криптографических ключей.
- `partner` — юридическое лицо или интегратор, имеющий договор и доступ к сервису.
- `tenant` — технический контур учёта партнёра в инфраструктуре сервиса.
- `application` — конкретное клиентское приложение партнёра, использующее API.
- `client_id` — идентификатор приложения партнёра для квотирования, аналитики и биллинга.
- `sandbox key` — тестовый ключ доступа для работы только в Sandbox.
- `production key` — ключ доступа к продуктивному API после завершения приёмки.
- `partner API key` — ключ, выданный партнёру для вызова публичных партнёрских API; в зависимости от контура может быть sandbox или production.

## Стандарт именования

- основной пользовательский метод во всех документах именуется `location refine`;
- в текстовых описаниях допускается русская формулировка `уточнение координат`;
- термины `refine` и `location correction` не используются как самостоятельные названия метода вне контекста конкретных SDK или внутренних сокращений.
