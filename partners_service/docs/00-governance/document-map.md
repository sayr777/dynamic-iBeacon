# Document Map

Документация разделена по контурам ответственности, чтобы продуктовые, технические, операционные и юридические материалы не дублировали друг друга.

## Распределение тем

- `01-product` содержит описание сервиса, дорожную карту и верхнеуровневые KPI.
- `02-architecture` фиксирует целевую архитектуру, сервисные контракты и границы интеграции с РНИС.
- `03-security` описывает модель угроз, управление ключами и требования compliance.
- `04-api` и `07-billing` содержат внешние API и OpenAPI-спецификации.
- `05-partner-integration` и `06-sandbox` закрывают практический контур подключения партнёров.
- `08-operations` описывает SLA, мониторинг, администрирование и поддержку.
- `09-commercial`, `10-finance`, `11-legal-and-public-sector` используются для коммерческого, финансового и договорного сопровождения.
- `12-enablement` содержит материалы для пилотов, презентаций и ответов партнёрам.

## Принцип актуализации

- тематические документы в `docs/` используются как рабочая и согласуемая версия по каждому контуру;
- при изменении API, SLA, тарифов или договорных условий обновляются соответствующие тематические файлы и связанная OpenAPI-спецификация.
