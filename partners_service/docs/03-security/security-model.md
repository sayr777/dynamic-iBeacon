# Security Model

Сервис построен по принципу доверенного распознавания: клиентское приложение и радиоэфир не получают постоянный идентификатор объекта и не владеют ключом дешифровки. Ключи и справочники остаются в защищённом контуре, а партнёр получает только минимально необходимый результат API-вызова.

## Базовые угрозы

- пассивное считывание эфира и попытки восстановить постоянный идентификатор;
- подбор или компрометация криптографического ключа;
- спуфинг радиоточек и подмена `radio_mac`;
- replay-атаки через повторную отправку старых payload;
- злоупотребление API и утечка технических данных через журналы.

## Контрмеры

- передача в эфир только динамических параметров;
- хранение ключей исключительно в KMS/HSM;
- проверка `timestamp`, `radio_mac`, аномального `rssi` и физически невозможных сценариев;
- минимизация ответа API и исключение PII из прикладного потока;
- централизованное логирование, audit trail и контроль доступа по ролям.
