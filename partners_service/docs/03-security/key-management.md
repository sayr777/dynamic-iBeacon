# Key Management

Ключи дешифровки хранятся в выделенном криптографическом контуре и не передаются в клиентские приложения, sandbox payload или интеграционные материалы партнёров.

## Правила управления ключами

- ключи создаются и хранятся в KMS/HSM;
- доступ к операциям дешифровки предоставляется только внутренним сервисам по сервисным ролям;
- плановая ротация выполняется по расписанию с контролируемым переходным периодом;
- аварийная ротация запускается при инциденте безопасности или подозрении на компрометацию;
- события ротации и смены прав доступа журналируются и доступны для аудита.

## Совместимость

Sandbox использует отдельные тестовые ключи. Production-ключи не копируются в тестовый контур и не используются для демонстрационных сценариев.
