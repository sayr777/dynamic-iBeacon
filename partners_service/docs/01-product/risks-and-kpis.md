# Risks And KPIs

## Ключевые KPI

- `P95 latency` основного API — менее 50 мс;
- доступность production API — не ниже 99.9%;
- точность дешифровки и сопоставления — не ниже 99.5%;
- время подключения нового партнёра — до 14 дней;
- MTTR по инцидентам критичности P1 — менее 30 минут.

## Основные риски

- устаревание текущей криптографической схемы и необходимость перехода на более современный режим;
- отказ BLE-меток или деградация батарей в региональной инфраструктуре;
- пиковая перегрузка API и биллинговых агрегаторов;
- регуляторные требования к логированию, хранению и аудиту технических данных.

## Митигации

- плановая модернизация криптографии и ротации ключей;
- инвентарный мониторинг и исключение невалидных объектов из маппинга;
- autoscaling, rate limiting и асинхронная обработка usage-событий;
- обезличивание логов, ограничение сроков хранения и формализованный аудит доступа.
