# SDK Guides

Клиентская интеграция поддерживает Android, iOS и Flutter. Во всех вариантах приложение выполняет BLE-сканирование, извлекает поля пакета и отправляет нормализованный payload в `Location Refine API`.

## Общие требования

- стабильная отправка `uuid`, `major`, `minor`, `rssi`, `radio_mac`, `timestamp`;
- синхронизированное системное время устройства;
- безопасное хранение API-ключа в клиентской конфигурации приложения или через защищённый backend-посредник партнёра;
- обработка временных ошибок сети и корректная телеметрия по `trace_id`.

## Платформенные особенности

- Android требует учёта разрешений BLE и фоновых ограничений;
- iOS требует корректной работы с CoreBluetooth и системными политиками энергосбережения;
- Flutter повторяет мобильную модель платформы и использует native-слои для BLE-сканирования.

## Рекомендуемый жизненный цикл вызова

1. Приложение фиксирует BLE-событие.
2. Выполняется локальная валидация обязательных полей.
3. Формируется JSON payload.
4. Payload отправляется в `POST /v1/location/refine`.
5. Результат используется в прикладной логике партнёра.
6. При ошибке сохраняются `trace_id`, код ответа и технический контекст вызова.
