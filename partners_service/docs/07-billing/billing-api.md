# Billing API

Партнёрский биллинговый API предоставляет доступ к профилю партнёра, списку `client_id`, текущему использованию и истории расчётных документов.

## Основные методы

- `GET /v1/partner/account` — профиль партнёра и активный тариф;
- `GET /v1/partner/clients` — список приложений и `client_id`;
- `POST /v1/partner/clients` — создание нового `client_id`;
- `GET /v1/billing/usage` — текущее использование и квоты;
- `GET /v1/billing/invoices` — история счетов и статусов.

## Пример сценария использования

1. Партнёр создаёт отдельные `client_id` для мобильных приложений или интеграционных потоков.
2. Через `GET /v1/billing/usage` отслеживает текущее потребление и приближение к лимиту.
3. Через `GET /v1/billing/invoices` получает расчётные документы за отчётный период.
4. При необходимости подключает вебхуки для событий квот и выставления счетов.

## Минимальные поля бизнес-ответов

- профиль партнёра: `partner_id`, `tier`, `status`, `sla`;
- `client_id`: идентификатор, метка, статус, даты создания и активности;
- usage: период, лимит, фактическое использование, перерасход, расчётная стоимость;
- invoice: идентификатор, период, сумма, статус, дата выставления.

## Правила использования

Все вызовы защищены партнёрским API-ключом. Доступ к данным ограничивается собственным партнёрским контуром без видимости чужих usage и расчётных данных.

Billing API и его поля должны соответствовать [openapi.yaml](C:\T1_GIT\ble-tag-jdy23-dynamic\partners_service\docs\07-billing\openapi\partner-billing\openapi.yaml).
