# Location Refine API

`POST /v1/location/refine` — основной публичный метод сервиса. Он принимает BLE-событие от клиентского приложения партнёра и возвращает уточнённую координату объекта, его идентификатор и оценку достоверности результата.

## Назначение метода

Метод используется в online-сценарии, когда клиентское приложение фиксирует BLE-пакет в зоне, где спутниковая навигация недостаточно точна. Сервис сопоставляет динамические идентификаторы с доверенным справочником и возвращает результат, пригодный для прикладной логики партнёра.

## HTTP-контракт

- Метод: `POST`
- Путь: `/v1/location/refine`
- Content-Type: `application/json`
- Аутентификация: `X-API-Key`
- Таймаут со стороны клиента: рекомендуется 3-5 секунд

## Структура запроса

Обязательные поля:

- `timestamp` — время фиксации события в формате `date-time`
- `uuid` — UUID BLE-пакета
- `major` — динамическая старшая часть идентификатора
- `minor` — динамическая младшая часть идентификатора
- `rssi` — уровень сигнала
- `radio_mac` — MAC-адрес радиомодуля

Опциональные поля:

- `gps_lat`
- `gps_lon`
- `gps_accuracy_m`

## Пример запроса

```json
{
  "timestamp": "2026-05-03T14:22:01.123Z",
  "uuid": "E2C56DB5-DFFB-48D2-B060-D0F5A71096E0",
  "major": 12345,
  "minor": 67890,
  "rssi": -62,
  "radio_mac": "AA:BB:CC:DD:EE:FF",
  "gps_lat": 54.1930,
  "gps_lon": 37.6175,
  "gps_accuracy_m": 15
}
```

## Пример ответа

```json
{
  "refined_lat": 54.193042,
  "refined_lon": 37.617581,
  "stop_id": "TUL_0042",
  "stop_name": "пр. Ленина, д. 12",
  "confidence": 0.94,
  "processing_time_ms": 38
}
```

## Семантика ответа

- `refined_lat`, `refined_lon` — итоговая координата, используемая партнёром в прикладной логике;
- `stop_id` — идентификатор объекта в доверенном справочнике;
- `stop_name` — человекочитаемое название объекта;
- `confidence` — оценка достоверности сопоставления в диапазоне от `0` до `1`;
- `processing_time_ms` — серверное время обработки запроса.

## Ограничения

- временное окно приёма события — до 3 секунд от `timestamp`;
- запросы с невалидным форматом BLE-параметров отклоняются;
- сервис не гарантирует успешное сопоставление для повреждённых, replay или spoofing-сценариев;
- при превышении лимитов доступа может возвращаться `429`.

## Версионирование

Изменения запроса и ответа синхронно фиксируются в этом документе и в [openapi.yaml](C:\T1_GIT\ble-tag-jdy23-dynamic\partners_service\docs\04-api\openapi\client-service\openapi.yaml). Изменение обязательных полей допускается только в рамках новой версии контракта.
