# Auth And Rate Limits

Доступ к публичному API осуществляется по заголовку `X-API-Key`. Ключ связан с партнёром, окружением, набором разрешённых методов и правилами квотирования.

## Контуры доступа

- `sandbox key` используется только в тестовом контуре;
- `production key` используется после завершения приёмки;
- `partner API key` относится к конкретному партнёру и может существовать в sandbox или production-варианте.

## Рекомендуемые заголовки

- `X-API-Key` — обязательный ключ доступа;
- `X-Client-Id` — рекомендуется для точного учёта приложений партнёра;
- `X-Request-Id` — опциональный корреляционный идентификатор со стороны партнёра.

## Ограничения и политики

- лимиты задаются на уровне партнёра и, при необходимости, на уровне `client_id`;
- превышение лимита приводит к ответу `429` и фиксируется в usage-контуре;
- попытки злоупотребления API могут приводить к временному ограничению или приостановке ключа.

## Политика безопасности

- ключи Sandbox и production не взаимозаменяемы;
- скомпрометированный ключ подлежит немедленной ротации;
- хранение production-ключа в клиентском приложении допускается только при соблюдении политики защищённой доставки конфигурации;
- для высокорисковых интеграций рекомендуется backend-proxy на стороне партнёра.
