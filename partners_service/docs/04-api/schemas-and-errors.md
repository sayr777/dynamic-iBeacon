# Schemas And Errors

## Базовые схемы

### `RefineRequest`

Содержит технические метаданные BLE-события и опциональные GPS-поля. Минимально допустимый запрос должен содержать `timestamp`, `uuid`, `major`, `minor`, `rssi` и `radio_mac`.

### `RefineResponse`

Возвращает:

- итоговую координату;
- идентификатор и название объекта;
- confidence;
- время обработки.

### `ErrorResponse`

Ошибки возвращаются в JSON-формате с машинным кодом, человекочитаемым описанием и `trace_id`, пригодным для диагностики в поддержке.

## Коды ошибок

| Код | Смысл | Типичная причина | Действие клиента |
|---|---|---|---|
| `400` | Неверный запрос | Ошибка схемы, формат поля, устаревший `timestamp` | Исправить payload и повторить |
| `401` | Нет доступа | Неверный или отозванный API-ключ | Проверить ключ и окружение |
| `403` | Доступ запрещён | Метод не разрешён для ключа или контура | Проверить права и профиль доступа |
| `429` | Превышен лимит | Rate limit или исчерпание квоты | Повторить позже, проверить usage |
| `500` | Внутренняя ошибка | Ошибка маршрутизации или обработки | Повторить с backoff |
| `503` | Временная недоступность | Плановые работы или деградация зависимости | Повторить с backoff |

## Рекомендации по обработке ошибок

- логировать `trace_id`, код ответа и время запроса;
- не ретраить ошибки валидации `400`;
- ретраить `429`, `500`, `503` с ограниченным backoff;
- разделять Sandbox и production-диагностику, чтобы не смешивать контуры.
