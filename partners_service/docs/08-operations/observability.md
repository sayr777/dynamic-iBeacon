# Observability

Мониторинг сервиса объединяет технические метрики, usage-аналитику, трейсы и журналы доступа. Отдельно ведутся dashboard-ы для Sandbox и production.

## Обязательные сигналы

- количество запросов и ошибки по кодам ответа;
- латентность основного API и внутренней дешифровки;
- аномалии по `rssi`, `radio_mac` и временным окнам;
- usage по `partner` и `client_id`;
- действия административного контура и доступ к журналам.

## Цели наблюдаемости

Набор метрик должен позволять одновременно отслеживать качество сервиса, корректность биллинга и события безопасности.

## Минимальные dashboard-ы

- API health и latency;
- usage и приближение к квотам;
- безопасность и аномалии запросов;
- административная активность и изменения справочников.
