# Contract Package

Договорный пакет оформляет коммерческий доступ партнёра к сервису и определяет технические, операционные и финансовые условия взаимодействия.

## Состав пакета

- основной договор;
- техническое приложение с описанием сервиса и API;
- SLA-приложение;
- коммерческое приложение с тарифами и биллингом;
- политика обработки технических метаданных.

## Ключевые положения

- границы ответственности сторон;
- порядок предоставления и прекращения доступа;
- правила использования ключей и SDK;
- ограничения на злоупотребление API и реверс-инжиниринг;
- порядок аудита и урегулирования инцидентов.

## Обязательные приложения к договору

1. Техническое описание сервиса и API.
2. SLA с метриками и классификацией инцидентов.
3. Коммерческое приложение с тарифом, квотой и overage.
4. Политика обработки технических метаданных.

## Рекомендуемые юридические акценты

- в базовом сценарии не обрабатываются персональные данные;
- ответственность поставщика ограничивается договорным контуром и согласованным объёмом услуг;
- ключи, справочники и криптографические материалы остаются в доверенном контуре исполнителя;
- доступ может быть приостановлен при компрометации ключа, злоупотреблении API или нарушении условий использования.
