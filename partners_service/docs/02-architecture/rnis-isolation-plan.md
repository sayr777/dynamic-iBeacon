# RNIS Isolation Plan

Изоляция сервисов дешифровки и справочников позволяет масштабировать партнёрский сервис без изменения ядра РНИС и без раскрытия чувствительных внутренних компонентов во внешний контур.

## Границы ответственности

- РНИС остаётся доверенным источником региональных справочников и части эксплуатационных данных.
- Partner Service отвечает за внешние API, аутентификацию партнёров, usage metering и биллинг.
- Управление ключами и доступом организуется через отдельный защищённый административный контур.

## План реализации

1. Выделить справочник объектов и дешифровку в независимые сервисы с формальными API.
2. Ограничить доступ к ним только через доверенные внутренние маршруты.
3. Добавить журналирование вызовов, версионирование справочников и контроль доступа.
4. Перевести внешние API на использование изолированных сервисов без прямой зависимости от внутренней логики РНИС.
