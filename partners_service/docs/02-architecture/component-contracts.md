# Component Contracts

## Внешний контракт

Клиентское приложение вызывает `POST /v1/location/refine`, передавая BLE-метаданные, временную метку, параметры сигнала и, при наличии, GPS-координаты. Ответ содержит уточнённую координату, `stop_id`, название объекта, confidence и время обработки.

## Внутренние контракты

- `Gateway -> Ingestion` передаёт уже аутентифицированный запрос с `partner_id`, `client_id`, `trace_id` и нормализованным payload.
- `Ingestion -> Decryption` передаёт проверенный набор полей `uuid`, `major`, `minor`, `rssi`, `radio_mac`, `timestamp`.
- `Decryption -> Registry` обращается к региональному справочнику для разрешения объекта и получения координат.
- `Decryption -> Usage Metering` публикует событие факта использования для последующей агрегации и биллинга.

## Совместимость

Все межсервисные сообщения должны быть версионированы. Изменение обязательных полей в запросах и событиях допускается только с обратной совместимостью или через новый номер схемы.
