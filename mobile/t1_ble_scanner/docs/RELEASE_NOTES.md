# T1 BLE Scanner — Release Notes

## v1.2.0 — 2026-05-17

### Исправления

#### 🔵 BLE-сканирование не восстанавливалось после долгой работы

**Симптом:** после ~30 минут работы нажатие кнопки «очистить» (корзина) приводило к тому, что радар переставал находить метки до перезапуска приложения.

**Причина:** Android (в т.ч. Xiaomi HyperOS) молча убивает `lowLatency` BLE-скан примерно через 30 минут — без вызова `onError` и без закрытия стрима. Флаг `_scanning` оставался `true`, но новые пакеты не приходили. После `clearDevices()` список пустел и не заполнялся снова.

**Решение — три слоя защиты:**

| Слой | Механизм |
|---|---|
| Подписка на `isScanning` | Мгновенно детектирует OS-kill и запускает рестарт |
| `onDone` в `scanResults.listen` | Ловит явное закрытие стрима ОС |
| `Timer(25 мин)` | Проактивный рестарт до порога ~30 мин |

Рестарт выполняется с задержкой 2 с (чтобы ОС освободила BLE-ресурсы) без потери состояния — список устройств и кэш дешифровки сохраняются.

**Дополнительно:** исправлен вторичный баг — `FlutterBluePlus.isScanning` является `BehaviorSubject` и при подписке сразу эмитирует `false`, что порождало бесконечный цикл рестартов каждые 2 с. Исправлено флагом `_restarting`, подавляющим ложные события на время запуска скана.

## v1.1.0 — 2026-05-17

### Новое

#### 📡 Реестр операторов UUID
- Новый экран **«Операторы UUID»** (кнопка в AppBar)
- Любой оператор — UUID, название, код и **цвет на радаре** (12 вариантов)
- Операторы из конфига добавляются автоматически при первом запуске
- Все записи редактируемы и удаляемы
- Данные сохраняются в `SharedPreferences` между сессиями

#### 🛑 Справочник остановок — упрощён
- Единый плоский список без разделения «по умолчанию / пользовательские»
- Все записи выглядят одинаково: карандаш + корзина у каждой
- Конфигурационные остановки не возвращаются после удаления (tombstone-механизм)

#### 🎨 Радар — цвета операторов
- Каждый оператор отображается точкой **своего выбранного цвета** + мягкое свечение
- Легенда упрощена: фиксированные записи для T1 resolved, T1, iBeacon, BLE

### Исправления

| Проблема | Решение |
|---|---|
| Зависание при старте | Дебаунс `notifyListeners` 100 мс, BLE-сканирование стартует сразу |
| Ложный диалог «Bluetooth выключен» | Ожидание стабильного состояния адаптера до 4 с |
| После нажатия «очистить» — беконы больше не находятся | `clearDevices()` теперь очищает кэш AES-дешифровки |
| В редакторе остановок кнопка корзины появлялась только после редактирования | Кнопки редактирования и удаления всегда видны у всех записей |

### Производительность

- **AES-128**: ключ разворачивается один раз на вызов дешифровки (было: на каждый блок) — ускорение ~5–10×
- **Изоляты Dart**: тяжёлые AES-вычисления вынесены в `compute()`, UI-поток не блокируется
- **Кэш TextPainter**: статичные подписи радара создаются один раз в `State`, не на каждый кадр

### Ориентация

- Экран зафиксирован в **портретной ориентации** (`android:screenOrientation="portrait"`)
