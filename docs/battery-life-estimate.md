# Оценка срока жизни батареи

## Платформа: nRF52810 (E73-2G4M04S1A)

## Исходные данные

| Параметр | Значение | Источник |
|---|---|---|
| Батарея `ER14505H-LD` номинальная ёмкость | `2700 мАч` | FANSO datasheet |
| Батарея консервативная доступная ёмкость | `2160 мАч` (80%) | инженерный запас |
| `nRF52810` deep sleep с `LFXO 32768 Гц` | `1.5 µА` | Nordic PS nRF52810 |
| `nRF52810` TX ток при `0 dBm` | `5.3 мА` | Nordic PS nRF52810 |
| Длительность одного TX события (3 канала) | `~1 мс` | BLE spec + Nordic |
| Интервал рекламы | `2 с` | проект |
| `MCP1700` ток покоя `Iq` | `~1 µА` | Microchip DS |
| Саморазряд `Li-SOCl₂` | `~1%/год` | FANSO spec |

## Расчёт среднего тока

### Фаза deep sleep (1999 мс из каждых 2000 мс)

```
I_sleep = (1.5 µА + 1.0 µА) × 1999/2000 = 2.498 µА
           nRF52810   MCP1700
```

### Фаза TX события (1 мс из каждых 2000 мс)

```
I_tx = 5.3 мА × 1 мс / 2000 мс = 2.65 µА
```

### Фаза AES + обновление параметров (5 мс каждые 300 000 мс = раз в 5 мин)

```
I_aes = 4 мА × 5 мс / 300 000 мс = 0.067 µА
```

### Итого

```
I_avg = 2.498 + 2.65 + 0.067 ≈ 5.2 µА
```

## Ресурс батареи

### Только нагрузочный ток

```
Life_load = 2160 мАч / 5.2 µА / 8760 ч/год = 47.5 лет
```

### С учётом саморазряда

```
Расход за год = 5.2 µА × 8760 ч + 2700 мАч × 1% = 45.6 + 27 = 72.6 мАч/год
Ресурс        = 2160 мАч / 72.6 мАч/год ≈ 30 лет
```

### Практический ресурс

`Li-SOCl₂` деградирует ускоренно при температурах выше `+50 °C`. С учётом реальной эксплуатации в диапазоне `-40…+65 °C`:

| Условия | Расчётный ресурс |
|---|---:|
| Комнатная температура | `~30 лет` |
| Уличная эксплуатация (−40…+65 °C) | `~15–20 лет` |
| **Проектный целевой срок** | **3 года** |
| **Запас по цели** | **×5–6** |

## Чувствительность к параметрам

| Интервал рекламы | Средний ток TX | Полный ток | Ресурс |
|---:|---:|---:|---:|
| `1 с` | `5.3 µА` | `8.8 µА` | `~23 лет` |
| **`2 с`** (базово) | **`2.65 µА`** | **`5.2 µА`** | **`~30 лет`** |
| `5 с` | `1.06 µА` | `3.6 µА` | `~40 лет` |

При любом практически разумном интервале цель `3 года` выполняется с многократным запасом.

## Сравнение платформ

| Платформа | Средний ток | Ресурс (с саморазрядом) |
|---|---:|---:|
| `ble-tag-jdy23` (статика, JDY-23) | `~50 µА` | `~5 лет` |
| `ble-tag-jdy23-dynamic` (JDY-23 + CW32L010) | `~22 µА` | `~10 лет` |
| **`ble-tag-jdy23-dynamic` (nRF52810)** | **`~5.2 µА`** | **`~30 лет`** |

## Варианты питания

### Вариант 2 — классическая первичная батарея (текущий выбор)

Средний ток устройства: **5.2 µА**

| Батарея | Химия | Ёмкость | Расч. ресурс | Практический |
|---|---|---:|---:|---:|
| `CR2032` (монета ∅20 мм) | Li-MnO₂ | 220 мАч | 4.6 лет | ~4 года |
| `CR2450` (монета ∅24.5 мм) | Li-MnO₂ | 620 мАч | 12 лет | ~8–10 лет |
| `ER14250` (1/2 AA, ∅14 мм) | Li-SOCl₂ | 1200 мАч | 20 лет | ~15 лет |
| **`ER14505H-LD`** (AA, **текущий**) | Li-SOCl₂ | 2700 мАч | **30 лет** | **~20 лет** |
| `CR123A` | Li-MnO₂ | 1500 мАч | 25 лет | ~12–15 лет |

**Методика расчёта** (на примере CR2032):
```
Нагрузка:     5.2 µА × 8760 ч/год   = 45.6 мАч/год
Саморазряд:   220 мАч × 1%/год      = 2.2 мАч/год
Итого:                                 47.8 мАч/год
Ресурс:       220 / 47.8             = 4.6 года
```

**Рекомендация по форм-фактору**:
- Компактная метка (≤ 40×40 мм): `CR2450` → 8–10 лет, цель ×3
- Промышленная метка (текущий корпус): `ER14505H` → 20 лет, цель ×7
- Минимальный размер при цели 3 года: `CR2032` → 4 года, цель ×1.3 (мало запаса)

### Вариант 1 — аккумулятор LiPo + солнечная панель

#### Состав

| Компонент | Пример | Цена | Примечание |
|---|---|---:|---|
| Солнечная панель 30×40 мм аморфный Si | Seeed/AliExpress | ~$1.00–1.50 | 2V / 3–5 мА при 500 лк |
| Микросхема сбора энергии | `BQ25504RGTR` (TI) | ~$2.50 | MPPT, Iq 488 нА |
| LiPo аккумулятор 100 мАч 3.7V | с защитной платой | ~$1.50 | буфер для тёмных периодов |
| LDO `MCP1700` 3.0V | тот же что в Вариант 2 | ~$0.25 | питание nRF52810 |
| **Доп. стоимость vs Вариант 2** | | **+$4.00–5.00** | |

#### Расчёт

**Среднесуточное потребление устройства:**
```
5.2 µА × 24 ч = 124.8 µАч/день = 0.125 мАч/день
```

**Саморазряд LiPo 100 мАч** (~3.6%/мес):
```
100 мАч × 3.6%/мес / 30 дней = 0.120 мАч/день  ← саморазряд ≈ нагрузке!
Суммарный расход в темноте: 0.125 + 0.120 = 0.245 мАч/день
```

**Ресурс без солнца** (только LiPo 100 мАч):
```
100 мАч / 0.245 мАч/день = 408 дней ≈ 1.1 года
```
→ LiPo без солнца **не** обеспечивает цель 3 года (LiPo саморазряд доминирует).

**Минимальная подзарядка для бесконечного ресурса:**
```
Нужно: 0.245 мАч/день
При 8 ч/день освещённости: 0.245 / 8 = 30.6 µА от панели
Панель 30×40 мм при 50 лк: ~30–50 µА → точка безубыточности ~20–30 лк
```

**При офисном освещении** (500 лк, 8 ч/день):
```
Генерация: ~400 µА × 8 ч = 3200 µАч/день
Потребление: 245 µАч/день
Коэффициент: ×13 → аккумулятор заряжен → бесконечный ресурс
```

#### Точка безубыточности

| Освещённость | Ток панели 30×40 мм | Генерация 8 ч/день | Баланс |
|---|---:|---:|---|
| 10 лк (тёмный коридор) | ~6 µА | 48 µАч/день | дефицит |
| **20 лк** (минимальный коридор) | **~15 µА** | **120 µАч/день** | ~равновесие |
| 100 лк (офис) | ~80 µА | 640 µАч/день | ×2.6 излишек |
| 500 лк (яркий офис) | ~400 µА | 3200 µАч/день | ×13 излишек |
| 10 000 лк (улица) | ~5 мА | 40 мАч/день | ×163 излишек |

#### Сравнение вариантов

| Критерий | Вариант 1 (LiPo + Solar) | Вариант 2 (ER14505H) |
|---|---|---|
| Ресурс при нормальном освещении | **бесконечный** | ~20 лет |
| Ресурс в полной темноте | ~1.1 года | ~20 лет |
| BOM | ~$10.38 (+$4 vs V2) | ~$6.38 |
| Сложность схемы | выше (BQ25504 + LiPo) | простая (LDO) |
| Надёжность | LiPo деградация ~2–3 года | нет движущихся частей |
| Применение | хорошо освещённые помещения, улица | производство, склад, любая темнота |

**Рекомендация**: для промышленного образца в условиях неконтролируемого освещения —
Вариант 2 (`ER14505H`, 20 лет) надёжнее и проще. Вариант 1 оправдан для
стационарных точек с гарантированным дневным светом.

## Ночной режим (Night Mode)

### Идея

Когда транспорт не работает, нет смысла отправлять iBeacon каждые 2 с.  
В ночные часы (например, 23:00–06:00) интервал пробуждения увеличивается с **2 с** до **60 с**.  
Параметры iBeacon (Major/Minor/MAC) всё равно меняются раз в 5 мин — детекция по `unix_time`, а не по счётчику циклов.

### Расчёт экономии

**Ночной режим: 23:00–06:00 = 7 ч ночь + 17 ч день**

| Период | Интервал | TX-вклад | Sleep | LDO | Итого |
|---|---|---:|---:|---:|---:|
| День (17 ч) | 2 с | 2.65 µА | 1.50 µА | 1.00 µА | **5.15 µА** |
| Ночь (7 ч) | 60 с | 0.09 µА | 1.50 µА | 1.00 µА | **2.59 µА** |

TX-вклад ночью: `5.3 мА × 1 мс / 60 000 мс = 0.09 µА` — практически ноль.

**Среднесуточный ток:**
```
I_avg = (5.15 × 17 + 2.59 × 7) / 24 = (87.55 + 18.13) / 24 = 4.40 µА
```

| Режим | Средний ток | Ресурс (ER14505H) | Ресурс (CR2450) |
|---|---:|---:|---:|
| Без ночного режима | 5.2 µА | ~20 лет | ~8 лет |
| **С ночным режимом** | **4.4 µА** | **~24 года** | **~10 лет** |
| Экономия | **−15%** | **+4 года** | **+2 года** |

### Конфигурация (tag_config.h)

```c
#define TAG_NIGHT_MODE_ENABLE        1
#define TAG_TIMEZONE_OFFSET_SEC      10800U   /* UTC+3 (Москва) */
#define TAG_NIGHT_START_SEC          (23U * 3600U)   /* 23:00 местного */
#define TAG_NIGHT_END_SEC            ( 6U * 3600U)   /* 06:00 местного */
#define TAG_NIGHT_WAKE_INTERVAL_SEC  60U              /* 1 мин ночью    */
```

### Почему детекция по unix_time, а не по cycle_count

При 60 с интервале ночью: `150 циклов × 60 с = 150 мин` — параметры бы не обновлялись 2.5 часа.  
Решение: `current_slot = unix_time / 300` — слот меняется ровно раз в 5 мин **независимо** от интервала пробуждения.
