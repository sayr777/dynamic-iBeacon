# Оценка срока жизни батареи

## Актуальная платформа: nRF52832 ([YJ-16013](../specs/YJ-16013-datasheet.pdf))

Текущий проект ориентирован на `nRF52832` в модуле [YJ-16013](../specs/YJ-16013-datasheet.pdf). Ниже расчёт среднего тока и срока жизни батареи именно для этого варианта. Для справки в конце оставлен короткий ориентир по `nRF52810`, потому что по потреблению он близок, но основным решением больше не является.

## Исходные данные для nRF52832 ([YJ-16013](../specs/YJ-16013-datasheet.pdf))

| Параметр | Значение | Источник |
|---|---|---|
| Батарея `ER14505H-LD` номинальная ёмкость | `2700 мАч` | FANSO datasheet |
| Консервативно доступная ёмкость | `2160 мАч` (80%) | инженерный запас |
| `nRF52832` deep sleep с `LFXO 32768 Гц` | `~1.5 µА` | Nordic Product Specification |
| `nRF52832` TX ток при `0 dBm` | `~5.3 мА` | Nordic Product Specification |
| Один advertising event (3 канала) | `~1 мс` | BLE timing estimate |
| Интервал пробуждения | `2 с` | текущая конфигурация |
| Обновление `UUID / Major / Minor / MAC` | `~5 мс` раз в `300 с` | оценка прошивки |
| `MCP1700` ток покоя | `~1.0 µА` | Microchip datasheet |
| Саморазряд `Li-SOCl₂` | `~1%/год` | FANSO datasheet |

## Расчёт потребления nRF52832 ([YJ-16013](../specs/YJ-16013-datasheet.pdf))

### Фаза deep sleep

В спящем режиме ток дают сам `nRF52832` и LDO:

```text
I_sleep = (1.5 µА + 1.0 µА) × 1999 / 2000
        = 2.498 µА
```

### Фаза BLE-передачи

Один advertising event передаётся примерно за `1 мс` и повторяется раз в `2 с`:

```text
I_tx = 5.3 мА × 1 мс / 2000 мс
     = 2.65 µА
```

### Фаза AES и обновления payload

Пересчёт параметров выполняется один раз на слот, то есть раз в `5 мин`:

```text
I_slot = 4 мА × 5 мс / 300000 мс
       = 0.067 µА
```

### Итог по среднему току

```text
I_avg = 2.498 + 2.65 + 0.067
      ≈ 5.2 µА
```

## Ресурс батареи для nRF52832 ([YJ-16013](../specs/YJ-16013-datasheet.pdf))

### Только по нагрузке

```text
Life_load = 2160 мАч / 0.005215 мА / 8760 ч
          ≈ 47.3 года
```

### С учётом саморазряда

```text
Нагрузка за год   = 5.2 µА × 8760 ч = 45.6 мАч/год
Саморазряд за год = 2700 мАч × 1%   = 27.0 мАч/год
Итого             = 72.6 мАч/год

Life_total = 2160 / 72.6
           ≈ 29.8 года
```

### Практический ресурс

Реальный срок службы ограничивается уже не потреблением SoC, а качеством батареи, температурой, утечками на плате и хранением:

| Условия | Практическая оценка |
|---|---:|
| Комнатная / лабораторная эксплуатация | `~25–30 лет` |
| Реальная уличная / промышленная | `~15–20 лет` |
| **Цель проекта** | **3 года** |
| **Запас по цели** | **×5–6** |

## Чувствительность к интервалу рекламы

| Интервал | TX-вклад | Полный средний ток | Практический вывод |
|---:|---:|---:|---|
| `1 с` | `5.3 µА` | `~8.8 µА` | запас по ресурсу всё ещё большой |
| **`2 с`** | **`2.65 µА`** | **`~5.2 µА`** | базовый режим проекта |
| `5 с` | `1.06 µА` | `~3.6 µА` | полезно, если приоритет только автономность |

Даже базовый режим `2 с` даёт многократный запас относительно цели `3 года`.

## Ночной режим

Если ночью увеличить интервал пробуждения, средний ток дополнительно снижается.

Пример для режима `23:00–06:00`, где ночью пробуждение идёт раз в `60 с`:

| Период | Интервал | Средний ток |
|---|---|---:|
| День | `2 с` | `~5.15 µА` |
| Ночь | `60 с` | `~2.59 µА` |
| **Среднесуточно** | — | **~4.4 µА** |

Это даёт ещё около `15%` экономии без изменения аппаратной части.

## Справочно: расчёт для nRF52810

При тех же `2 с`, той же батарее и том же `MCP1700` итог получается практически тем же:

```text
I_avg(nRF52810) ≈ 5.2 µА
```

Именно поэтому переход на `nRF52832` в модуле [YJ-16013](../specs/YJ-16013-datasheet.pdf) не ухудшает автономность, но даёт более удобную и актуальную платформу для текущего проекта.

## Что подтвердить на образце

```text
Цель: средний ток < 10 µА на реальной плате от батареи
Метод: шунт 10 Ом + осциллограф / логгер, усреднение не меньше 60 с
Блокирующий дефект: ток > 20 µА — искать утечки, неверный CBUF или лишнюю периферию
```
