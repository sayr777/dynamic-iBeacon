# Алгоритм динамической идентификации

## Задача

Каждые `5 мин` метка публикует новые `Major` и `Minor` (и опционально суффикс `MAC`). Наблюдатель не может отследить метку по этим значениям. Сервер, знающий ключ, однозначно определяет метку по паре `(Major, Minor)` за время `O(N)`, где `N` — количество меток в регионе (до `10 000`).

## Параметры

| Параметр | Размер | Описание |
|---|---|---|
| `TAG_ID` | `16 бит` | статичный уникальный номер метки, `0...65535` |
| `KEY` | `16 байт` | общий секретный ключ региона (одинаков на всех метках и на сервере) |
| `SLOT_DURATION` | `300 с` | длительность одного временного слота (настраиваемо) |
| `slot` | `32 бита` | номер текущего слота: `unix_time / SLOT_DURATION` |

## Вычисление параметров метки

```
block[0..1]  = TAG_ID (big-endian, 2 байта)
block[2..5]  = slot   (big-endian, 4 байта)
block[6..15] = 0x00   (10 байт заполнения)

out[0..15] = AES-128-ECB-Encrypt(KEY, block)

major      = (out[0] << 8) | out[1]     # 16 бит
minor      = (out[2] << 8) | out[3]     # 16 бит
mac[3]     = out[4] | 0xC0              # locally administered, random
mac[4]     = out[5]
mac[5]     = out[6]
```

Первые `3` байта `MAC` фиксированы: `E5:00:00` (или любой другой OUI с установленным `LA`-битом). Последние `3` байта меняются вместе с `Major/Minor`.

## Идентификация на сервере

Для каждого принятого пакета `(major_rx, minor_rx)`:

```python
slot_now = int(unix_time()) // SLOT_DURATION

for s in [slot_now - 1, slot_now, slot_now + 1]:   # окно ±5 мин
    for tag_id in range(0, NUM_TAGS):               # перебор всех меток
        block = tag_id.to_bytes(2, 'big') + s.to_bytes(4, 'big') + bytes(10)
        out = AES128_encrypt(KEY, block)
        major_c = (out[0] << 8) | out[1]
        minor_c = (out[2] << 8) | out[3]
        if major_c == major_rx and minor_c == minor_rx:
            return tag_id  # метка найдена
return None  # не найдена
```

С `10 000` метками и `3` слотами — `30 000` операций `AES-128` ≈ `< 1 мс` на современном сервере.

## Вероятность коллизии

Вероятность того, что две разные метки в одном слоте дадут одинаковые `(Major, Minor)`:

```
P_collision ≈ N² / 2³² ≈ 10000² / 4294967296 ≈ 0.0023%
```

При коллизии сервер получит два совпадения — обе метки можно вернуть как кандидатов. На практике контекст (местоположение сканера, последняя известная позиция метки) позволяет разрешить неоднозначность.

## Свойства безопасности

- **Неотслеживаемость**: без знания `KEY` наблюдатель не может связать пакеты одной метки между слотами.
- **Аутентичность**: подобрать `(tag_id, slot)` под произвольную пару `(Major, Minor)` без `KEY` — задача инверсии `AES-128`.
- **Минимальность**: используется только 32 бита из 128-битного вывода `AES`; остальные 80 бит отброшены. Это не ослабляет безопасность: злоумышленник всё равно не может инвертировать `AES` зная только 32 выходных бита.
- **Общий ключ**: один ключ на весь регион. Компрометация одного устройства раскрывает все метки региона. Для повышения безопасности можно использовать иерархию ключей: `device_key = AES(master_key, tag_id)`.

## Управление ключами

- `KEY` генерируется один раз для региона: [`server/keygen.py`](../server/keygen.py).
- `KEY` прошивается в каждый `STM32L031` при производстве.
- `KEY` хранится на сервере в защищённом хранилище.
- `KEY` **никогда не передаётся** по воздуху.

## Иерархия ключей (расширенный вариант)

Если нужна изоляция компрометации отдельного устройства:

```
master_key   — хранится только на сервере (16 байт)
device_key_i = AES128(master_key, tag_id_i || 0x00[14])  — для каждой метки
```

Тогда компрометация одного `device_key_i` не раскрывает другие метки.

## Пример вычисления

```
TAG_ID = 42      → 0x002A
slot   = 17520   → 0x00004470  (unix_time = 5256000 / 300 = 17520)
KEY    = 2B7E151628AED2A6ABF7158809CF4F3C

block  = 002A 00004470 00000000000000000000

AES128(KEY, block) → например: 3A7F C1B2 E49D A1 ...

major = 0x3A7F  → 14975
minor = 0xC1B2  → 49586
mac   = E5:3A:E4:9D:A1:XX  (последние 3 байта из out[4..6])
```

## Тестирование алгоритма

```bash
# Симуляция одного цикла метки
python server/lookup.py --simulate --tag-id 42 --slot 17520

# Идентификация принятого пакета
python server/lookup.py --major 3A7F --minor C1B2
```
