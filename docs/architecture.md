# Архитектура изделия

## Назначение

Динамическая BLE-метка для автономной передачи `iBeacon`-посылок, в которых `Major`, `Minor` и суффикс `MAC`-адреса меняются каждые `5 мин`. При знании секретного ключа сервер однозначно восстанавливает статичный идентификатор метки по паре `(Major, Minor)`.

## Основные узлы

- хост-контроллер `STM32L031F6P6` (альтернатива: `STM32L010F4P6`)
- готовый BLE-модуль `JDY-23`
- батарея `FANSO ER14505H-LD` с выводами под пайку
- диод Шоттки + буферный конденсатор `47...100 µF`
- корпус `AK-W-70-4`

## Логика работы

### Цикл передачи (каждые 2 с)

1. `RTC` будит `STM32L031` из `Stop mode`.
2. Контроллер инкрементирует счётчик циклов `cycle_count`.
3. Если `cycle_count < CYCLES_PER_SLOT` — параметры не меняются, переход к п. 5.
4. Если `cycle_count >= CYCLES_PER_SLOT`:
   - `cycle_count = 0`, `slot++`;
   - вычислить `block = tag_id || slot || 0x00_pad`;
   - `out = AES128(KEY, block)`;
   - `major = out[0:2]`, `minor = out[2:4]`, `mac_suffix = out[4:7]`;
   - пометить флаг `params_changed = 1`.
5. Включить питание `JDY-23` через GPIO.
6. Ждать `50 мс` (модуль стартует и успевает передать `1` пакет).
7. Если `params_changed`:
   - отправить `AT+MAJOR`, `AT+MINOR`;
   - отправить `AT+RST`;
   - ждать `500 мс` (перезагрузка и передача пакета с новыми параметрами);
   - `params_changed = 0`.
8. Выключить питание `JDY-23` через GPIO.
9. `STM32L031` входит в `Stop mode` до следующего `RTC`-будильника.

### Первый запуск

При первом запуске от батареи контроллер не знает текущего времени. `unix_time` должен быть установлен при производственной калибровке через `UART` (команда конфигурационного утилиты). После этого `RTC` ведёт отсчёт автономно.

## Питание

```
ER14505H-LD -> Schottky -> CBUF 47...100 µF -> VCC_MAIN (3.6 В)
VCC_MAIN -> STM32L031 VDD   (прямое питание, 1.65...3.6 В)
STM32L031 GPIO_JDY_PWR -> P-MOS -> JDY-23 VCC
```

Почему `JDY-23` питается через GPIO хоста:

- `JDY-23` активен только `~80 мс` из каждых `2000 мс`;
- это снижает среднее потребление `JDY-23` в `25` раз по сравнению с постоянным включением;
- `STM32L031` в `Stop mode` потребляет `≈ 0.4 µA` и почти не влияет на ресурс батареи.

## Потребление в номинальном цикле

| Фаза | Ток | Длительность | Вклад за цикл |
|---|---:|---:|---:|
| `STM32L031` Stop mode | `0.4 µA` | `~1870 мс` | `0.748 µAh/цикл` |
| `STM32L031` активный | `~3 мА` | `~10 мс` | `0.008 µAh/цикл` |
| `JDY-23` активный (`100 мс`) | `~15 мА` | `~80 мс` | `0.333 µAh/цикл` |
| Схема питания (диод, MOSFET) | `~1 µA` | постоянно | `0.556 µAh/цикл` |
| **Итого** | — | **2000 мс** | **≈ 1.65 µAh/цикл** |

Средний ток: `≈ 3.0 µA`. Это расчётный ориентир — реальное значение нужно проверить измерением.

## Контроль времени

- `RTC STM32L031` работает от кварца `32768 Гц` в `Stop mode`;
- drift кварца типового качества: `±20 ppm` = `±1 с/день`;
- за год накопится `≈ ±6 мин` дрейфа;
- сервер проверяет текущий слот ±1 (окно `±5 мин`) — допустимо;
- опционально: периодическая синхронизация через `BLE` или при обслуживании.

## Принятые архитектурные решения

| Решение | Обоснование |
|---|---|
| `AES-128 ECB` вместо `HMAC-SHA256` | меньше Flash и RAM, аппаратная поддержка на `STM32` |
| хост управляет питанием `JDY-23` | снижение среднего тока `JDY-23` в `25×` |
| хранение `unix_time` в `RTC` | не требует GPS и сетевого стека |
| шаг `2 с` вместо `1 с` | ещё в `2×` снижает потребление `JDY-23` |
| `JDY-23` без собственной прошивки | простота, отсутствие необходимости в SDK |
