# Архитектура изделия

## Назначение

Динамическая BLE-метка для автономной передачи `iBeacon`-посылок, в которых `Major`, `Minor` и суффикс `MAC`-адреса меняются каждые `5 мин`. При знании секретного ключа сервер однозначно восстанавливает статичный идентификатор метки по паре `(Major, Minor)`.

## Основные узлы

- хост-контроллер `STM32L010F4P6` (или аналог, см. `docs/bom.md`)
- готовый BLE-модуль `JDY-23`
- батарея `FANSO ER14505H-LD` с выводами под пайку
- диод Шоттки + буферный конденсатор `47...100 µF`
- корпус `AK-W-70-4`

## Ключевое инженерное решение по питанию

`JDY-23` **работает непрерывно** на своём внутреннем цикле рекламы `2 с`.  
`STM32L010` **спит в Stop mode** и просыпается только раз в `5 мин` чтобы обновить параметры через AT-команды.

### Почему нельзя включать/выключать JDY-23 каждые 2 с

При управлении питанием JDY-23 через GPIO (P-MOS):

```
15 мА × 80 мс / 2000 мс = 600 µА средний ток от JDY-23 → ~6 месяцев батареи
```

При непрерывной работе JDY-23 с внутренним интервалом `2 с`:

```
~15...20 µА средний ток от JDY-23 → >10 лет батареи
```

Вывод: **JDY-23 должен рекламировать на внутреннем таймере**, а не на внешнем управлении питанием.

## Логика работы

### Нормальный цикл (каждые 5 мин)

1. `RTC` будит `STM32L010` из `Stop mode`.
2. `slot++`, `cycle_count++`.
3. Вычислить `block = tag_id || slot || 0x00_pad`.
4. `out = AES128(KEY, block)` → `major`, `minor`, `mac_suffix`.
5. Отправить `AT+MAJOR{XXXX}`, `AT+MINOR{XXXX}` в `JDY-23` через `UART`.
6. Отправить `AT+RST` — `JDY-23` перезагружается и начинает рекламировать с новыми параметрами.
7. `STM32L010` входит в `Stop mode` на следующие `5 мин`.

### Первый запуск

При первом запуске от батареи `unix_time` должен быть установлен при производственной калибровке через `UART`. После этого `RTC` ведёт отсчёт автономно.

## Питание

```
ER14505H-LD → Schottky → CBUF 47...100 µF → VCC_MAIN (≈3.5 В)
VCC_MAIN → STM32L010 VDD  (прямое питание, допустимо 1.65...3.6 В)
VCC_MAIN → JDY-23 VCC     (прямое питание, всегда включён)
```

P-MOS ключ питания `JDY-23` **не требуется** — модуль включён постоянно.

## Бюджет потребления

| Узел | Режим | Ток | Доля времени | Средний вклад |
|---|---|---:|---:|---:|
| `JDY-23` | реклама `2 с` интервал | `≈17 µА` | 100% | `17 µА` |
| `STM32L010` | `Stop mode` с `RTC` | `0.29 µА` | ~99.8% | `0.29 µА` |
| `STM32L010` | активный (AT-обновление) | `2 мА` | ~0.2% | `4 µА` |
| Схема питания | утечки | `<0.5 µА` | 100% | `0.5 µА` |
| **Итого** | — | — | — | **≈ 22 µА** |

Средний ток `~22 µА`. Срок службы батареи `ER14505H-LD 2700 мАч`:

```
Нагрузочный ток:  22 µА × 8760 ч = 193 мАч/год
Саморазряд:       2700 мАч × 1%  =  27 мАч/год
Итого:            220 мАч/год
Ресурс:           2700 / 220 = ~12 лет
```

Целевой срок `3 года` обеспечивается с **четырёхкратным запасом**.

## Контроль времени

- `RTC STM32L010` работает от кварца `32768 Гц` в `Stop mode` (`0.29 µА`);
- дрейф типового кварца: `±20 ppm` → `≤ 6 мин/год`;
- сервер проверяет текущий слот `±1` → дрейф компенсируется;
- опционально: синхронизация при плановом обслуживании через `UART`.

## Архитектурные решения

| Решение | Обоснование |
|---|---|
| `JDY-23` работает непрерывно | Внутренний сон модуля (≈17 µА) эффективнее MCU power cycling (600 µА) |
| MCU просыпается только раз в 5 мин | Нет необходимости считать каждые 2 с — JDY-23 тайминг автономный |
| `AES-128 ECB` | Меньше Flash и RAM чем `HMAC-SHA256`, достаточно для 16 KB Flash |
| Нет P-MOS ключа | Упрощение схемы, снижение BOM |
| `STM32L010F4P6` | Самый дешёвый STM32 с RTC + UART + Stop 0.29 µА |
