# Архитектура изделия

## Назначение

Динамическая BLE-метка для автономной передачи `iBeacon`-посылок, в которых `Major`, `Minor` и `MAC`-адрес меняются каждые `5 мин`. Сервер, знающий ключ, **однозначно восстанавливает статичный идентификатор метки** по паре `(Major, Minor)`.

## Выбранная платформа

**`EBYTE E73-2G4M04S1A`** — программируемый BLE-модуль на `nRF52810`.

Один чип заменяет два (JDY-23 + MCU). Полный контроль над каждым рекламным пакетом: `Major`, `Minor`, `MAC`, интервал, TX power.

Обоснование выбора: [`platform-comparison.md`](platform-comparison.md)

## Основные узлы

- BLE-модуль `E73-2G4M04S1A` (SoC `nRF52810`)
- LDO `MCP1700T-3002E/TT` — `3.0 В`
- батарея `FANSO ER14505H-LD` с выводами под пайку
- буферный конденсатор `100...220 µF`
- корпус `AK-W-70-4`

## Логика работы

### Основной цикл (каждые 2 с)

1. `RTC` (`LFXO 32768 Гц`) будит `nRF52810` из `System OFF / deep sleep`.
2. `cycle_count++`.
3. Если `cycle_count >= CYCLES_PER_SLOT` — выполнить обновление параметров (п. 4–7).
4. `cycle_count = 0`, `slot++`.
5. `block = tag_id || slot || 0x00_pad`.
6. `out = AES128(KEY, block)` → `major = out[0:2]`, `minor = out[2:4]`, `mac[3:6] = out[4:7]`.
7. Обновить `advertising data` и `MAC` в стеке BLE.
8. Запустить один `advertising event` (3 канала, ~1 мс).
9. Перейти в `deep sleep` до следующего `RTC`-будильника (`+2 с`).

### Первый запуск

`unix_time` устанавливается при производственной прошивке через `SWD`. После этого `RTC` ведёт отсчёт автономно в `deep sleep`.

## Питание

```
ER14505H-LD (+3.6 В)
    │
   [CBUF 100...220 µF]
    │
   [MCP1700T-3002E/TT] → 3.0 В → E73 VCC
    │
   GND
```

`MCP1700` обеспечивает стабильные `3.0 В` и защищает `nRF52810` от верхнего предела батарейного напряжения `3.6 В`.

## Бюджет потребления

| Фаза | Ток | Длительность | Средний вклад |
|---|---:|---:|---:|
| Deep sleep с RTC | `1.5 µА` | `~1999 мс / 2000 мс` | `1.50 µА` |
| TX событие (3 канала) | `5.3 мА` | `~1 мс / 2000 мс` | `2.65 µА` |
| Обработка AES + обновление params | `~4 мА` | `~5 мс / 300 000 мс` | `0.07 µА` |
| LDO `MCP1700` ток покоя | `~1 µА` | постоянно | `1.00 µА` |
| **Итого** | — | — | **≈ 5.2 µА** |

Ресурс батареи `ER14505H-LD 2700 мАч`:

```
Нагрузка:     5.2 µА × 8760 ч = 45.6 мАч/год
Саморазряд:   2700 мАч × 1%   = 27.0 мАч/год
Итого:        72.6 мАч/год
Ресурс:       2700 / 72.6 = ~37 лет → ограничен саморазрядом
Практически:  ~15–20 лет
```

Целевой срок `3 года` обеспечивается с **запасом ×5–6**.

## Связь с ble-tag-e73

Аппаратная часть совпадает с [`ble-tag-e73`](../../ble-tag-e73): те же модуль, LDO, батарея, корпус. Прошивка отличается:

| | `ble-tag-e73` | **ble-tag-jdy23-dynamic** |
|---|---|---|
| Режим BLE | Observer + Broadcaster | Только Broadcaster |
| Параметры рекламы | фиксированные | AES-ротация каждый слот |
| Протокол | двунаправленный БНСО | одностороннее beacon |
| SoftDevice | S132 | S112 (только Broadcaster) |
