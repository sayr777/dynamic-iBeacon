# Ограничения и риски

## Функциональные ограничения

- `JDY-23` не поддерживает смену `MAC`-адреса через `AT+MAC` во всех ревизиях прошивки — перед серией нужно подтвердить наличие этой команды;
- если `AT+MAC` не поддерживается, `MAC` остаётся статичным, что ослабляет приватность (наблюдатель сможет отследить метку по `MAC`, но не идентифицировать её без ключа);
- нет контроля батареи — хост не измеряет напряжение питания;
- хост не имеет двусторонней BLE-связи — только передача;
- точность `RTC` зависит от качества кварца: за год может набежать `±6 мин` дрейфа.

## Электрические риски

- `JDY-23` управляется через MOSFET-ключ: нужно подобрать MOSFET с малым `Vgs(th)` совместимым с напряжением GPIO хоста;
- при повторных включениях конденсатора в `JDY-23` может возникнуть бросок тока через MOSFET — нужно проверить на реальном образце;
- при несовпадении логических уровней `STM32L031` (3.3 В) и `JDY-23` (3.3 В) — уровни совместимы, но нужно проверить конкретную ревизию платы.

## Производственные риски

- прошивка `STM32L031` требует SWD-программатора и производственного jig — усложняет линию по сравнению с `ble-tag-jdy23`;
- `TAG_ID` должен быть уникальным на весь регион — нужна база учёта;
- `KEY` должен оставаться секретным — компрометация одного изделия позволяет прочитать `KEY` из Flash (без защиты Read-out Protection);
- разные партии `JDY-23` могут вести себя по-разному при управлении питанием через MOSFET.

## Риски безопасности

- `STM32L031` без настроенного `RDP Level 1/2` позволяет прочитать `KEY` через `SWD` — обязательно включить Read-out Protection перед финальной прошивкой;
- один `KEY` на весь регион: компрометация любого устройства раскрывает все метки региона — для критичных применений использовать иерархию ключей (описано в `docs/algorithm.md`);
- слот длиной `5 мин` и `10 000` меток: вероятность коллизии `~0.002%` — приемлемо.

## Механические риски

- те же, что в `ble-tag-jdy23`: компаунд в RF-зоне, батарея перед антенной;
- дополнительно: жёсткая пайка двух плат может передавать механические напряжения.

## Рекомендации

- включить `RDP Level 1` на `STM32L031` перед финальной сборкой;
- проверить реакцию `JDY-23` на управление питанием через MOSFET на первой партии;
- проверить наличие `AT+MAC` на используемой ревизии `JDY-23`;
- использовать кварц с точностью `±10 ppm` для лучшей стабильности времени.
