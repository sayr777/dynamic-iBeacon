# Производственная схема

## Целевой маршрут

1. Закупить модули `E73-2G4M04S1A` одной ревизии.
2. Собрать плату: `E73` + `MCP1700` + обвязка + тест-поинты `SWD`.
3. Прошить `nRF52810` через `SWD` с конфигурацией: `TAG_ID`, `KEY`, `unix_time`.
4. Активировать `APPROTECT` (включено в финальной прошивке).
5. Проверить BLE-рекламу: `Major/Minor` должны меняться раз в `5 мин`.
6. Припаять батарею.
7. Проверить запуск от батареи и рекламу.
8. Установить в корпус, залить компаундом (не в антенную зону).
9. Финальный BLE-тест и контроль тока.

## Прошивка nRF52810

Каждый модуль прошивается индивидуально:

```
Конфигурируемые параметры на каждое изделие:
  TAG_ID    — уникальный номер (0..65535), записать в реестр меток
  KEY       — региональный ключ (одинаковый для всех меток региона)
  unix_time — текущее время UTC в момент прошивки
```

Команды (nRF5 SDK + nrfjprog):

```bash
# Прошить SoftDevice S112 (один раз на линии)
nrfjprog --program s112_nrf52_7.3.0_softdevice.hex --chiperase

# Прошить приложение с TAG_ID=42
make TAG_ID=42 flash

# Проверить что APPROTECT активирован
nrfjprog --readregs
```

## Синхронизация времени

`unix_time` записывается в момент прошивки. `nRF52810` хранит его в `GPREGRET2` (сохраняется при `System OFF`) и инкрементирует каждые `2 с` при пробуждении по `RTC`.

Дрейф `LFXO`: `±20 ppm` → `≤ 6 мин/год`. Сервер проверяет слот `±1`.

## Отличие от ble-tag-e73 и ble-tag-jdy23

| Этап | `ble-tag-jdy23` | `ble-tag-e73` | **ble-tag-jdy23-dynamic** |
|---|---|---|---|
| AT-настройка модуля | да | нет | нет |
| Прошивка SWD | нет | да | **да** |
| Уникальные данные | Major/Minor | tag_id | **TAG_ID + KEY** |
| Этапов на линии | 1 | 1 | **1** |
| BLE-реклама начинается | сразу | после команды | **сразу после старта** |

## Контрольные точки качества

До пайки батареи:
- `nrfjprog --ids` видит чип;
- BLE-реклама видна сканером;
- `Major/Minor` меняются раз в `5 мин`;
- `MAC` меняется вместе с `Major/Minor`;
- ток deep sleep `< 3 µА`, средний ток `< 10 µА`.

После пайки батареи:
- запуск от батареи без зависания;
- нет нагрева `MCP1700`;
- напряжение `VDD_3V0` в норме (`2.95...3.05 В`).

После сборки в корпус:
- антенна `E73` ориентирована к пластику;
- дальность не деградировала;
- компаунд не попал в RF-зону.

## Блокирующие дефекты

- `APPROTECT` не активирован перед заливкой;
- ток `> 20 µА` в среднем — лишний потребитель на плате;
- `Major/Minor` не меняются;
- сервер не идентифицирует метку;
- нет рекламы после пайки батареи.
