# Производственная схема

Платформа: **[YJ-16013](../specs/YJ-16013-datasheet.pdf) (nRF52832)**.

## Целевой маршрут

1. Закупить модули [YJ-16013](../specs/YJ-16013-datasheet.pdf) одной партии.
2. Собрать плату: [YJ-16013](../specs/YJ-16013-datasheet.pdf) + `MCP1700` + `BAT54` + обвязка + тест-поинты `SWD/RESET`.
3. Прошить `nRF52832` через `SWD` (J-LINK / nRF52 DK): `SoftDevice S112` + приложение с `TAG_ID`, `KEY`, `unix_time`.
4. Активировать `APPROTECT` (включено в финальной прошивке).
5. Проверить BLE-рекламу: `Major/Minor` меняются раз в `5 мин`, `MAC` меняется вместе.
6. Припаять батарею.
7. Проверить запуск от батареи и рекламу.
8. Установить в корпус, залить компаундом (не в антенную зону).
9. Финальный BLE-тест и контроль тока (цель `< 10 µА` средний).

## Прошивка nRF52832 (nRF5 SDK + nrfjprog)

Каждый модуль прошивается индивидуально с уникальными параметрами:

```
Конфигурируемые параметры на каждое изделие:
  TAG_ID    — уникальный номер (0..65535), записать в реестр меток
  KEY       — секретный AES-128 ключ по выбранной схеме provisioning
  unix_time — текущее время UTC в момент прошивки
```

Команды:

```bash
# 1. Прошить SoftDevice S112 v7.3.0 (один раз на линии)
nrfjprog --program s112_nrf52_7.3.0_softdevice.hex --chiperase --reset

# 2. Прошить приложение (TAG_ID уникален для каждой метки)
make TAG_ID=42 TAG_UNIX_TIME=$(date +%s) flash

# 3. Убедиться что APPROTECT активирован
nrfjprog --readregs | grep APPROTECT

# 4. Сброс и старт
nrfjprog --reset
```

Альтернатива через `west flash` (nRF Connect SDK):

```bash
west flash --runner nrfjprog -- --program s112_nrf52_7.3.0_softdevice.hex --chiperase
west flash --runner nrfjprog
```

## Синхронизация времени

`unix_time` записывается в момент прошивки через `TAG_INITIAL_UNIX_TIME` в `tag_config.h`.  
`nRF52832` хранит его в `NRF_POWER->GPREGRET2` (сохраняется при `System OFF + RTC`)  
и инкрементирует каждые `2 с` при пробуждении.

Дрейф `LFXO`: `±20 ppm` → `≤ 6 мин/год`. Сервер проверяет слот `±1` для компенсации.

## Производственные особенности

| Параметр | Значение |
|---|---|
| Интерфейс прошивки | `SWD` |
| Доступ к `RESET` | обязателен |
| SoftDevice | `S112` |
| Индивидуальные данные | `TAG_ID`, `KEY`, `unix_time` |
| Защита чтения | `APPROTECT` |
| Производственный реестр | обязателен для каждой метки |

## Контрольные точки качества

**До пайки батареи** (питание от программатора 3.3 В):
- `nrfjprog --ids` видит чип и показывает `nRF52832`;
- BLE-реклама видна сканером (nRF Connect / LightBlue);
- `Major/Minor` меняются раз в `5 мин` (`300 с`);
- `MAC` меняется вместе с `Major/Minor`;
- ток deep sleep `< 3 µА`, средний ток `< 10 µА`;
- сервер `server/lookup.py` идентифицирует метку по `Major/Minor`.

**После пайки батареи:**
- запуск от батареи без зависания;
- нет нагрева `MCP1700`;
- напряжение `VDD_3V0` в норме (`2.95...3.05 В`).

**После сборки в корпус:**
- антенна [YJ-16013](../specs/YJ-16013-datasheet.pdf) ориентирована к пластику;
- дальность не деградировала (сравнить до/после сборки);
- компаунд не попал в RF-зону.

## Блокирующие дефекты

| Дефект | Причина |
|---|---|
| `APPROTECT` не активирован перед заливкой | ключ `KEY` будет доступен через SWD |
| Средний ток `> 20 µА` | лишний потребитель на плате, проверить CBUF тип |
| `Major/Minor` не меняются | ошибка в `TAG_CYCLES_PER_SLOT` или `unix_time = 0` |
| Сервер не идентифицирует метку | неверный `KEY` или `TAG_ID` не в реестре |
| Нет рекламы после пайки батареи | батарея разряжена, неверная полярность `BAT54` |
